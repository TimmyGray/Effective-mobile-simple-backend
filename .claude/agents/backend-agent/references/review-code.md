# Review Backend Code

## Outcome

Produce a high-signal backend review focused on correctness, security, and convention alignment.

## Context

Use `docs/CONVENTIONS.md`, `docs/SECURITY.md`, and `docs/RELIABILITY.md` as baseline criteria.
Pay special attention to centralized authorization checks and `401` versus `403` response handling.

## Scope

In scope: backend diffs, API contracts, access-control flow, migration safety, and test sufficiency.
Out of scope: stylistic nits without operational or maintainability impact.
