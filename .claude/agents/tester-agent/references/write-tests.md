# Write Tests

## Outcome

Create or update tests that prove correctness of backend auth/authz behavior and prevent regressions.

## Context

The project baseline expects pytest. Use `docs/RELIABILITY.md` and `docs/CONVENTIONS.md`
to target boundary validation, status code semantics, and policy enforcement outcomes.

## Scope

In scope: backend unit/integration/API tests tied to changed behavior.
Out of scope: unrelated full-suite rewrites and cosmetic-only test refactors.
