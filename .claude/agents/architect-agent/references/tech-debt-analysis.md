# Analyze Technical Debt

## Outcome

Produce prioritized technical debt recommendations with clear rationale and sequencing.

## Context

Use `docs/exec-plans/tech-debt-tracker.md`, `docs/QUALITY_SCORE.md`, and `docs/PLANS.md` to align
recommendations with roadmap phases and current quality baseline.

## Scope

In scope: architecture debt, reliability debt, security debt, and sequencing guidance.
Out of scope: proposing roadmap items disconnected from current project goals.
