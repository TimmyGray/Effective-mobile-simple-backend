# Planning Memory System

## Outcome

Maintain lightweight planning memory across sessions: recent priorities, recurring blockers,
and observed velocity hints that improve subsequent planning decisions.

## Context

Persist memory in planning artifacts under `docs/exec-plans/` when explicitly requested.
Use current tracker status and quality baseline as objective signals.

## Scope

In scope: planning-state continuity and trend tracking.
Out of scope: storing sensitive data or undocumented side-channel state.
