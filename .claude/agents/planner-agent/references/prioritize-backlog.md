# Prioritize Backlog

## Outcome

Return a ranked backlog ordering with rationale tied to product goals, risk, and execution dependencies.

## Context

Use `docs/PRODUCT_SENSE.md`, `docs/PLANS.md`, and `docs/exec-plans/tech-debt-tracker.md`.
Auth/authz correctness and security are top-level risk reducers for this project.

## Scope

In scope: ranking tasks, splitting oversized tasks, and identifying blockers.
Out of scope: rewriting product strategy without explicit user direction.
